Summary of Findings

The experiment demonstrated the difference between hard links and symbolic links in Linux. Hard links share the same inode as the original file and remain usable even if the original file is deleted. Symbolic links have their own inode and only store the path to the original file. Once the original file is removed, the symbolic link becomes invalid. The 'ls -li' and 'stat' commands were used to examine inode numbers and file metadata throughout the experiment.
